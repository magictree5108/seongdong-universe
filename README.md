# NEO 성동

성동구 공공데이터를 3D 의미 연결망 + 온톨로지로 시각화한 Streamlit 앱입니다.
팔란티어 Gotham의 조사(investigation) UX에서 영감을 받았습니다.

[디딤(Didim)](../didim) 프로젝트가 수집·색인한 성동구 고시공고·자치법규·
사전컨설팅(감사원·자체감사기구) 선례 2,671건을 우주의 성단처럼 배치하고,
그 위에 규칙 기반으로 추출한 개체 레이어(담당 부서 31·법령 참조 79·동네 10)를
얹었습니다.

## 핵심 개념 (팔란티어 오마주)

| 개념 | 구현 |
|------|------|
| **온톨로지** | 문서에서 부서·법령·동네 개체를 추출해 타입 있는 노드(다이아몬드)·엣지로 승격 |
| **상황판** | 검색하면 관련 자료가 카메라 앞으로 끌려나오고 우측 패널에 카드로 정렬 |
| **AI 검색** | Claude가 질의를 행정용어로 확장(마음건강→심리상담·정신건강)하고, 어휘 일치 후보 중 실제 관련 자료만 선별 |
| **Object 360** | 노드 선택 시 속성·연결 개체·유사 문서를 보여주는 상세 패널. 연결 버튼으로 그래프를 계속 피벗 |
| **성좌(constellation) 뷰** | 선택한 노드와 그 연결만 밝히고 배경은 성운처럼 가라앉힘 |
| **케이스 파일** | 조사 중 발견한 자료를 핀으로 모아 마크다운 리포트로 내보내기 |
| **계기판 디테일** | 분류 배너(OPEN DATA), 하단 SYS 상태바, 모노스페이스 타이포 |

## 실행 방법

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
streamlit run app.py
```

브라우저에서 http://localhost:8501 접속. 데스크톱/와이드 화면을 권장합니다.
3D 화면은 마우스 드래그(회전)·스크롤(줌) 외에 우상단 ＋/－/⟲ 버튼으로도
확대·축소할 수 있습니다.

데이터(`data/`)는 이미 빌드되어 저장소에 포함돼 있으므로 위 실행만으로 바로
동작합니다 — 디딤 백엔드나 원본 데이터가 없어도 됩니다.

### AI 검색 활성화 (권장)

`.streamlit/secrets.toml`(git 제외됨)에 Anthropic API 키를 넣으면 검색이
2단계 AI 파이프라인으로 동작합니다:

```toml
ANTHROPIC_API_KEY = "sk-ant-..."
```

1. **질의 확장** — Claude(기본 `claude-haiku-4-5`)가 질의를 연관 행정용어로
   확장 (예: 마음건강 → 심리상담·정신건강)
2. **어휘 후보 수집** — 확장 키워드가 실제로 제목·본문에 등장하는 문서만
   후보로 수집 (공백 무시 매칭)
3. **AI 선별** — Claude가 후보 중 주제가 실제로 관련된 자료만 관련도순 반환

키가 없으면 원 질의의 어휘 일치 검색으로만 동작합니다(확장·선별 없음).
모델은 `NEO_LLM_MODEL` 환경변수로 바꿀 수 있습니다.

## 데이터 재빌드

디딤 프로젝트의 로컬 색인(`~/didim/data/index/*.meta.json`)이 갱신되었을 때:

```bash
python data_pipeline/build.py --didim-index-dir ~/didim/data/index
```

- 임베딩: 문자 n-gram 해싱 TF-IDF (외부 모델·API 불필요, `data_pipeline/embedder.py`)
- 3D 좌표: 순수 numpy PCA(SVD)
- 의미 연결망: 코사인 유사도 k-최근접 이웃
- 개체 추출: 정규식·사전 기반 (담당부서 `(담당: ○○과)` 패턴, 법령 `「…법/조례」` 인용,
  성동구 행정동 이름 매칭) — LLM 없이 결정적으로 동작

## 구조

```
neo-seongdong/
├── app.py                  # Streamlit 앱 (3D 시각화 + 검색 + Object 360 + 케이스 파일)
├── data_pipeline/
│   ├── embedder.py          # 해싱 n-gram TF-IDF 임베더 (빌드·검색 공용)
│   └── build.py             # 디딤 색인 → 노드/엣지/온톨로지/임베딩 빌드
├── data/                    # 빌드 산출물 (저장소에 커밋됨, 약 14MB)
│   ├── nodes.json           # 문서+개체 노드 (3D 좌표, 온톨로지 링크 포함)
│   ├── edges.json           # [i, j, 가중치, 타입(sim|dept|law|place)]
│   ├── embeddings.npy       # 노드별 1024차원 벡터 (검색용)
│   ├── idf.npy              # 검색 질의 임베딩에 쓰는 IDF 벡터
│   └── meta.json            # 빌드 통계
└── .streamlit/config.toml   # 다크 테마
```

## Streamlit Community Cloud 배포

1. 이 저장소를 GitHub에 푸시
2. https://share.streamlit.io 에서 새 앱 생성 → 이 저장소·`app.py` 선택
3. `requirements.txt`가 자동 인식되어 별도 설정 없이 배포됩니다

## 고지

이 프로젝트는 성동구가 공개한 행정데이터를 재구성한 시각화 데모입니다.
개체(부서·법령·동네)는 규칙 기반 자동 추출로 오추출이 있을 수 있고, 검색
결과의 관련성은 문자 기반 통계적 유사도로 계산된 것으로 내용의 정확성·최신성·
적법성을 보증하지 않습니다. 정확한 내용은 각 카드의 원문 링크에서 확인하십시오.
